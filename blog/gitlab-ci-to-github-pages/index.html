折腾记：把 Next.js 个人主页从 GitLab CI 移植到 GitHub Pages 的踩坑实录

2026-07-14

本以为"在 GitLab 编写代码、自动构建，然后一键部署到 GitHub Pages 展示"是一个半小时就能搞定的简单任务。

然而，在打通 "GitLab CI（加工厂） ➡️ GitHub Pages（展示柜）" 链路的过程中，我结结实实地踩了四个大坑。以下是我的血泪排障记录，希望帮大家少走弯路。

---

坑 1：GitLab Runner 权限不足（Docker-in-Docker 报错）

* 现象：流水线刚启动，Docker 守护进程就报错崩溃：

```text

mount: permission denied (are you root?)

Cannot connect to the Docker daemon at tcp://docker:2375.

```

* 原因：自建的 GitLab Runner 默认没有开启特权模式，导致嵌套的 Docker 容器无法挂载系统安全目录。

* 解法：修改宿主机上的 /etc/gitlab-runner/config.toml，在 [runners.docker] 下面将特权模式开启：

```toml

privileged = true

```

---

坑 2：样式丢失，网页白屏（Jekyll 屏蔽 _next）

* 现象：网页部署成功，但打开全是白屏，查看控制台，发现 _next/static 下的所有 CSS 和 JS 文件全部 404。

* 原因：GitHub Pages 默认用 Jekyll 引擎解析静态网页，而 Jekyll 会自动忽略任何下划线 _ 开头的文件夹（包括 Next.js 核心的 _next）。

* 解法：必须在部署根目录下生成一个空文件 .nojekyll，强行禁用 Jekyll 引擎。

---

坑 3：通配符 * 漏掉了点（隐藏）文件

* 现象：明明在脚本里写了 touch gh-pages/.nojekyll，推送上去后依然 404，发现该文件根本没被提交。

原因：传统的 cp -r out/ gh-pages/ 拷贝命令，其中的 * 默认无法匹配到以 . 开头的隐藏文件（如 .nojekyll）。

* 解法：改用末尾带点号的拷贝命令：

```bash

cp -r out/. gh-pages/

```

---

坑 4：GitLab Pages 本地静态目录"套娃"

* 现象：部署在 GitLab Pages 备份站点的网页也 404 了。

* 原因：在脚本里直接运行了 cp -r out/ public/。但由于 Next.js 源码中原本就有一个 public 文件夹，导致 cp 命令自作聪明地把整个 out 拷贝到了 public 的内部（变成了 public/out/index.html）。

* 解法：拷贝前，先无情地强制清理掉原有目录：

```bash

rm -rf public && cp -r out/ public/

```

---

💡 终极版 .gitlab-ci.yml 部署脚本

在搞定上述问题，并保证 GITHUB_TOKEN 变量名对齐后，我最终磨砺出了这一套能完美兼顾代码体检（Lint）、构建、并同时双发部署到 GitLab & GitHub 的极简配置：

image: node:20

stages:
  - lint
  - build
  - deploy

cache:
  key:
    files:
      - package-lock.json
    prefix: $CI_COMMIT_REF_SLUG
  paths:
    - node_modules/
    - .next/cache/

lint:
  stage: lint
  script:
    - npm ci
    - npm run lint
  only:
    - main

build:
  stage: build
  script:
    - npm ci
    - npm run build
  artifacts:
    paths:
      - out/
    expire_in: 1 hour
  only:
    - main

pages:
  stage: deploy
  script:
    - rm -rf public
    - cp -r out/ public/
  artifacts:
    paths:
      - public/
  only:
    - main

deploy-github:
  stage: deploy
  script:
    - git clone --depth 1 https://baobanxian:${GITHUB_TOKEN}@github.com/baobanxian/baobanxian.github.io.git gh-pages
    - rm -rf gh-pages/*
    - cp -r out/. gh-pages/
    - touch gh-pages/.nojekyll
    - cd gh-pages
    - git config user.name "baobanxian"
    - git config user.email "2793406287@qq.com"
    - git add .
    - git diff --quiet && git diff --staged --quiet || git commit -m "Deploy $(date '+%Y-%m-%d %H:%M:%S')"
    - git push origin main
  only:
    - main
