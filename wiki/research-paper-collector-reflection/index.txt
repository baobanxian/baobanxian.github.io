12:Tb28,<h2>我的原话</h2>
<p>> [张方宇] 今天我们在课堂上一起完成了 <code>research-paper-collector</code> 技能的搭建。整个过程包括：定义 SKILL.md 的工作流、编写 <code>collect_papers.py</code> 和 <code>check_papers.py</code> 脚本、创建 <code>check-site.sh</code> 零依赖检查脚本，最后把这些集成到网站中。</p>
<p>> [张方宇] 我特别注意到数据路径的问题——脚本里用了绝对路径 <code>/public/data/papers.json</code>，但实际项目根目录在 <code>/Users/baobanxian/my-portfolio</code>，如果不改成相对路径就会写出到系统根目录。这个 bug 是在运行检查脚本时才暴露的。</p>
<p>> [张方宇] 零依赖的 <code>check-site.sh</code> 让我印象很深。它只用 bash 和 perl（macOS 自带），不需要装任何包，适合放在 CI 的第一个步骤做快速健康检查。</p>
<h2>Agent 总结</h2>
<p>根据对话记录，Agent 完成了以下工作：</p>
<p>1. 创建 <code>.opencode/skills/research-paper-collector/SKILL.md</code>，包含工作流、输入参数和安全规则</p>
<p>2. 创建 <code>references/topics.md</code>，提供中文主题到 arXiv 查询词的映射</p>
<p>3. 编写 <code>scripts/collect_papers.py</code>，调用 arXiv API 获取论文，以 id 去重</p>
<p>4. 编写 <code>scripts/check_papers.py</code>，验证必需字段、重复 id、URL 格式</p>
<p>5. 运行收集脚本，成功获取 10 篇 AI Agent 相关论文</p>
<p>6. 修复 <code>check_papers.py</code> 中 URL scheme 只允许 https 的断言——arXiv API 返回的是 http 链接</p>
<p>7. 将脚本的 OUTPUT 路径从绝对路径改为基于 <code>__file__</code> 的相对路径</p>
<p>8. 更新 <code>app/papers/page.tsx</code>，从 <code>public/data/papers.json</code> 读取并展示论文列表</p>
<p>9. 编写 <code>scripts/check-site.sh</code>，零依赖 bash 脚本，覆盖必要页面、站内链接、RSS XML 检查</p>
<h2>我的反思</h2>
<p>这次实践让我意识到几个关键点：</p>
<p>1. <strong>路径问题容易在"最后一公里"暴露</strong>。脚本在本地跑没问题，但部署到 CI 或不同机器就可能出问题。以后写脚本应该优先用相对于脚本位置的路径。</p>
<p>2. <strong>外部 API 的返回格式不能假设</strong>。arXiv API 返回的 URL 是 http 而非 https，这提醒我们在写验证逻辑时要对实际数据做探查，而不是凭想象写断言。</p>
<p>3. <strong>零依赖工具链的价值</strong>。bash 写的检查脚本可以在任何 POSIX 环境运行，不依赖 node_modules，适合做 CI 快速门禁。</p>
